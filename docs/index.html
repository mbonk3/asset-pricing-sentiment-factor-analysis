Research Question

What drives the premium in anomaly portfolios?

This thesis sits at the intersection of behavioral finance and quantitative asset pricing. The central question is whether investor sentiment — isolated from macroeconomic noise — carries additional explanatory power for anomaly returns beyond established risk factors.

Does investor sentiment — once purified of macroeconomic confounds — help explain or predict the returns of well-documented anomaly (factor-based) portfolios?

Econometrics

Methodology

The analysis employs robust econometric techniques throughout, including Newey-West HAC standard errors, PCA for dimensionality reduction, and Benjamini-Hochberg FDR correction for multiple testing.

1

Sentiment Orthogonalization

Raw sentiment indicators conflate macroeconomic conditions with behavioral psychology. To isolate the pure behavioral channel, each sentiment indicator $S_t$ is regressed on six macroeconomic growth rates:

Orthogonalization Regression
$$S_t = \alpha + \sum_{k=1}^{6} \beta_k \,\Delta^{12} X_{k,t} + \varepsilon_t$$

The residuals $\varepsilon_t$ are the orthogonalized (behavioral) sentiment, stripped of macroeconomic confounds. All macro variables are transformed to 12-month growth rates for stationarity:

  • Real variables (IP, Consumption, Employment): $\Delta^{12} X_t = \ln(X_t) - \ln(X_{t-12})$
  • Nominal variables (CPI): $\Delta^{12} X_t = (X_t - X_{t-12}) / X_{t-12}$

Estimation: OLS with Newey-West HAC standard errors (4 lags).

Diagnostics: R², Variance Inflation Factor (VIF < 10), Breusch-Pagan heteroskedasticity test, Durbin-Watson autocorrelation test, Jarque-Bera normality test.

2

Factor Model Testing

Each anomaly portfolio is tested against three progressively comprehensive asset pricing models. A factor is considered a robust anomaly only if its alpha is statistically significant at the 5% level across all three models simultaneously.

CAPM
$$R_{it} - R_{ft} = \alpha_i + \beta_{i,MKT}(R_{mt} - R_{ft}) + \varepsilon_{it}$$
Fama-French 3-Factor
$$R_{it} - R_{ft} = \alpha_i + \beta_{MKT}(R_{mt} - R_{ft}) + \beta_{SMB}\,SMB_t + \beta_{HML}\,HML_t + \varepsilon_{it}$$
Fama-French 5-Factor
$$R_{it} - R_{ft} = \alpha_i + \beta_{MKT}(R_{mt} - R_{ft}) + \beta_{SMB}\,SMB_t + \beta_{HML}\,HML_t + \beta_{RMW}\,RMW_t + \beta_{CMA}\,CMA_t + \varepsilon_{it}$$

Benchmark-adjusted returns $\tilde{R}_{it}$ — the pure anomaly component after removing systematic risk exposures — are computed as FF5 residuals and used in the subsequent sentiment analysis.

3

Sentiment-Conditional Analysis

Each month is classified into a sentiment regime via tercile splits on the orthogonalized sentiment index. The middle tercile is excluded to sharpen the contrast between extreme regimes.

  • Low Sentiment: Bottom 33% of orthogonalized sentiment
  • High Sentiment: Top 33% of orthogonalized sentiment

The central test is the High-Minus-Low return differential:

High-Minus-Low Test
$$HML_{i,s} = \bar{R}_i^{High} - \bar{R}_i^{Low}, \qquad t_{NW} = \frac{HML_{i,s}}{\hat{SE}_{NW}(HML_{i,s})}$$

Tested separately for Long (D10), Short (D01), and Long-Short (D10−D01) portfolio legs, with Newey-West t-statistics (3 lags).

Significance thresholds: * 10% (|t| > 1.645)    ** 5% (|t| > 1.96)    *** 1% (|t| > 2.576)

+

Supplementary Analyses

  • Principal Component Analysis: Applied to standardized sentiment indicators to identify a common behavioral factor and assess collinearity. Run separately for four sample periods (Full, Long, Modern, Post-2000).
  • Correlation Analysis: Pairwise Pearson correlations between sentiment indicators with Benjamini-Hochberg FDR correction at the 5% level across multiple sample windows.
  • VIF Diagnostics: Variance Inflation Factor computed for each macro predictor in every orthogonalization regression (concern threshold: VIF > 10).
  • Alpha Survival: Proportion of anomaly factors retaining significant alphas as the model expands from CAPM to FF3 to FF5.

Reproducibility

Code & Repository Structure

The analysis is fully reproducible from a single pipeline script. The codebase is modular, configuration-driven, and includes checkpointing so long runs can be resumed.
